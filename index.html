NDVI de Baena (Córdoba)

Este proyecto analiza la evolución del índice de vegetación NDVI en explotaciones agrícolas de Baena (Córdoba) a partir de imágenes satelitales Sentinel-2. El objetivo fue crear una visualización interactiva que permita evaluar la actividad vegetal de cada parcela a lo largo del tiempo, facilitando la detección de anomalías o comportamientos atípicos.

Se utilizaron datos abiertos del SIGPAC de la Junta de Andalucía, filtrando más de 1.000 recintos agrícolas de los polígonos catastrales 1 al 8. Se excluyeron aquellas parcelas menores a 1.000 m² para centrarse en unidades productivas significativas. A partir de estas geometrías se generaron archivos GeoJSON que sirvieron como base para la visualización.

Las bandas B04 (rojo) y B08 (infrarrojo cercano) de Sentinel-2 se procesaron para calcular el NDVI en cinco fechas distintas. Estos valores se integraron como atributos espaciales en cada recinto y se visualizaron como capas raster mediante Leaflet.js. Además, se automatizó con Python la generación de gráficos NDVI temporales para cada parcela, y se programó un visor que permite:

  • Filtrar por tipo de cultivo (olivar, frutales, etc.).
  • Buscar por ID de recinto.
  • Visualizar estadísticas: media, desviación típica, tendencia y valores extremos.
  • Detectar automáticamente anomalías estacionales.

El resultado es una herramienta web interactiva que integra procesamiento de imágenes, análisis geoespacial y visualización dinámica del territorio, demostrando el potencial del código abierto para el monitoreo agrícola.

Clasificación automática del terreno y NDVI con Python

¿Es posible analizar imágenes satelitales y clasificar el terreno usando solo código? En este proyecto demuestro que sí: con Python puro, procesé imágenes multiespectrales de Sentinel-2 para generar indicadores ambientales y mapas temáticos sin depender de software SIG tradicional.

El objetivo era construir una herramienta ligera, replicable y accesible para automatizar el análisis de vegetación (NDVI) y la segmentación del paisaje mediante algoritmos no supervisados.

El resultado es un flujo de trabajo autónomo que convierte datos en información útil para la toma de decisiones ambientales, combinando ciencia de datos con teledetección de código abierto.

🧠 ¿Qué hace el script?

  • Lee imágenes multibanda (.tiff) de Sentinel-2 desde una carpeta local.
  • Genera una imagen RGB del territorio en colores naturales.
  • Calcula el índice NDVI y lo exporta como PNG y GeoTIFF.
  • Clasifica automáticamente el terreno mediante K-Means.
  • Exporta los resultados como imágenes visuales y archivos georreferenciados.

📂 Datos de entrada

  • Bandas B02, B03, B04 y B08 en formato .tiff.
  • Imágenes descargadas de SentinelHub o Copernicus Browser.

Visor interactivo por CCAA

Este visor permite representar visualmente una serie de indicadores territoriales en España a nivel de comunidades autónomas. El objetivo fue desarrollar una interfaz simple e interactiva que permita experimentar con mapas temáticos y escalas de color basadas en datos cargados manualmente.

El sistema se ha creado con Leaflet.js para la visualización geoespacial, TailwindCSS para el diseño de la interfaz y html2canvas para exportar el mapa como imagen. Los valores se almacenan directamente en un archivo GeoJSON y están organizados en categorías temáticas como agricultura, energía o aspectos sociales.

Entre sus funcionalidades se incluyen:

  • Selección de categoría e indicador desde un panel lateral.
  • Tooltip dinámico al pasar el ratón con el valor correspondiente.
  • Escala de color adaptativa con leyenda vertical.
  • Cabecera sincronizada con el indicador activo.
  • Botón para descargar el mapa generado como imagen PNG.

Nota: Los datos utilizados no provienen de fuentes oficiales y fueron generados y/o aproximados exclusivamente con fines de prueba y aprendizaje. Este visor no debe interpretarse como una herramienta de análisis técnico ni emplearse con fines profesionales o divulgativos.

Estimación de CO₂ equivalente tras el incendio de Las Hurdes

Este proyecto evalúa el impacto del incendio forestal ocurrido en mayo de 2023 en la comarca de Las Hurdes (Cáceres), a partir del análisis de imágenes Sentinel-2 y datos LiDAR del PNOA. Se delimita el área afectada mediante índices espectrales multitemporales y se compara la estructura vertical de la vegetación antes y después del incendio.

El objetivo es realizar una estimación aproximada del CO₂ equivalente que podría capturarse en caso de reforestar la superficie afectada, utilizando modelos simplificados y supuestos técnicos estándar sobre biomasa aérea.

Las imágenes fueron usadas para calcular el índice dNBR (Normalized Burn Ratio Difference), útil para identificar zonas afectadas por incendios según el cambio espectral. A partir de este índice se generó un mapa de severidad:

Mapa de severidad del incendio (dNBR)

Según esta clasificación, se estima que la superficie afectada alcanza aproximadamente 10.700 hectáreas.

Datos LiDAR – Zona quemada

Datos LiDAR – Zona no quemada

Los datos LiDAR se emplearon para generar el CHM (Canopy Height Model), comparando la altura media del dosel entre zonas afectadas y no afectadas. Se observa una pérdida notable de biomasa aérea en la zona incendiada.

Estimación de CO₂ equivalente capturable

Suponiendo una reforestación completa con Pinus pinaster a razón de 1.100 árboles por hectárea (marco de plantación de 3×3 m), y una biomasa aérea de 63 kg por árbol, se estima una captura teórica de:

  • 1.100 árboles × 63 kg = 69.300 kg de biomasa por hectárea
  • Contenido de carbono: 69.300 × 0,5 = 34.650 kg C
  • Conversión a CO₂eq: 34.650 × 3,67 = 127,17 t CO₂eq/ha

Escalado a 10.700 hectáreas → ≈1.360.670 toneladas de CO₂eq

Nota: Esta estimación representa un escenario hipotético máximo, sin considerar regeneración natural, limitaciones ecológicas, variabilidad de especies ni factores de degradación del suelo. Su uso es exclusivamente demostrativo.
